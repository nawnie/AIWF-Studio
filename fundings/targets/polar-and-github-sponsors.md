# Polar.sh + GitHub Sponsors

Polar: https://polar.sh/

GitHub Sponsors docs: https://docs.github.com/en/sponsors

## Fit

Strong immediate setup. This does not require a grant board and can turn the roadmap into fundable issues.

Best angle:

- Fund specific engineering outcomes.
- Let users and companies sponsor the local AI work they want.
- Convert large roadmap items into scoped issues.

## What You Need

- GitHub Sponsors profile.
- Polar account connected to GitHub.
- Public roadmap issues.
- Clear labels such as `funding`, `sponsor`, `good first fund`, `engine`, `training`, `video`, `privacy`.
- `FUNDING.yml` in `.github/` once profile URLs are ready.

## Suggested Fundable Issues

1. Harden ProcessSupervisor engine split.
2. Publish Wan/GGUF local video benchmark package.
3. Finish Training tab public docs and safety validation.
4. Add local model operation receipts and rollback.
5. Build first-run setup wizard for local models and optional engines.
6. Create privacy/security docs for local network exposure.

## Example Roadmap Issue Text

Title: Fund worker isolation for video, chat, and training engines

AIWF Studio is moving heavy local AI work into supervised workers so image generation, Wan video, chat, and training can coexist without crashing the main UI or fighting over GPU memory. Funding this issue supports implementation, tests, documentation, and demo workflows for the worker boundary.

Deliverables:

- Worker start/stop/cancel reliability.
- Structured logs and result paths.
- GPU tenant switch checks.
- Tests for process cleanup.
- Documentation and demo.

## Setup Checklist

- Create GitHub Sponsors profile.
- Create Polar profile.
- Decide monthly sponsor tiers.
- Add funding links to README.
- Add fundable roadmap issues.
- Add `.github/FUNDING.yml` after URLs are confirmed.

## Caveats

Do not add funding links until the profiles exist. Keep funded issue scopes specific enough that a sponsor can understand what their money unlocks.
